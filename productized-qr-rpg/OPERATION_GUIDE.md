# OPERATION GUIDE

## 기본 운영 흐름

1. 운영자는 `/admin`에 접속해 참가자/미션/교환소 상태를 확인합니다.
2. 학생은 `/`에서 이름과 조를 입력해 캐릭터를 만듭니다.
3. QR을 찍으면 `/mission/:code` 또는 `/hidden/:code`로 들어갑니다.
4. 미션 완료 버튼을 누르면 서버가 중복 여부와 잠금 여부를 확인한 뒤 보상을 지급합니다.
5. 이미 완료한 QR이면 "이미 완료한 미션" 상태로 안내합니다.
6. 관리자는 필요한 경우 달란트/장비를 수동 조정합니다.

## 동명이인 처리

같은 이름+조가 이미 있으면 새 캐릭터를 바로 만들지 않습니다. 학생은 입장코드를 입력하거나 선생님에게 확인해야 합니다.

운영 권장:

- 명찰 또는 조별 종이에 입장코드 4자리를 적어줍니다.
- 이름을 잘못 입력한 경우 관리자가 이름을 수정합니다.

## 현장 장애 대응

QR이 안 찍힐 때:

- 휴대폰 기본 카메라를 사용합니다.
- QR 출력물이 너무 작거나 흐리면 `/admin/print/qr`에서 다시 출력합니다.
- 직접 URL을 입력해 접속할 수 있습니다.

보상이 중복된 것 같을 때:

- `/admin`의 중복 완료 기록을 확인합니다.
- 필요하면 달란트 또는 장비를 수동 회수합니다.

인터넷이 불안정할 때:

- 학생에게 같은 버튼을 여러 번 누르지 않게 안내합니다.
- 서버는 같은 QR 중복 보상을 막지만, 화면 반영은 지연될 수 있습니다.

휴대폰 없는 학생:

- 선생님 휴대폰 또는 조장 휴대폰으로 학생 이름을 선택해 진행합니다.

전체 초기화:

- `/admin` 하단의 데이터 초기화 기능을 사용합니다.
- 운영 중 실수 방지를 위해 `RESET` 확인 문구가 필요합니다.
