# 이번 수련회 사용용: 전신갑주를 입어라

> 이 폴더는 이번 수련회에서 실제 배포하고 운영할 독립 앱입니다. 교회 납품·판매용 제품은 `../delivery/`에서 별도로 관리합니다.

2박 3일 중고등부 수련회용 전신갑주 RPG입니다. 학생 20~30명이 4~6명씩 파티를 만들고 장비를 모은 뒤, 마지막 날 열리는 `THE WAR`에서 파티 전원이 공동 QR을 스캔하고 각자의 강점에 맞는 파트를 맡아 6가지 미션에 도전합니다.

## 학생 진행 흐름

1. 이름과 조 이름을 입력해 입장합니다.
2. 같은 조 이름을 입력한 학생 4~6명이 한 파티로 모입니다.
3. 파티장이 명단을 확정하면 장비 뽑기가 바로 열립니다.
4. 각자 뽑은 장비는 개인 인벤토리에 보관됩니다.
5. 마지막 날 `THE WAR`가 열리면 파티원 모두가 각자의 휴대폰으로 같은 QR을 스캔합니다.
6. 화면에서 `스캔 완료 N/N`을 확인하며 마지막 파티원을 기다립니다.
7. 전원 스캔이 끝나면 합체 연출과 함께 파티 장비·전투력 계산 결과가 공개됩니다.
8. 각자 자신 있는 파트를 1~2개 선택합니다. 6개 파트를 모두 한 명씩 맡고, 모든 파티원이 최소 1개를 맡으면 미션과 보스가 열립니다.

파티 명단을 확정하기 전에는 학생과 파티장 모두 직접 탈퇴하고 조 이름을 다시 입력할 수 있습니다. 확정 뒤에는 탈퇴하거나 파티원을 바꿀 수 없습니다. 이름·조 편성 오류는 교사에게 요청하세요. 교사만 THE WAR 첫 QR 스캔 전까지 관리자 화면에서 비상 재개방할 수 있으며, 스캔이 시작되면 명단은 영구 잠금됩니다.

## 핵심 기능

- 이름 + 조 이름으로 로그인하고 같은 정보로 자동 이어하기
- 최초 파티 생성자를 파티장으로 지정
- 최소 4명, 최대 6명으로 파티 명단 확정
- 명단 확정 직후 1회·2회·3회 장비 뽑기 이용
- 홈에서 내 캐릭터, 개인 장비 현황, 인벤토리, 장비 전투력 확인
- 파티에서 파티원 정보와 각자의 개인 전투력 확인
- 마지막 날에만 열리는 `THE WAR`와 파티 전원 공동 QR 체크인
- 전원 스캔 완료 시 합체 연출과 파티 장비·전투력 스냅샷 공개
- 6개 고유 파트 중 한 사람당 1~2개를 직접 선택하는 역할 배분
- 파트 선택과 완료 상태를 파티원 전체 화면에 실시간 동기화
- 6개 파트 배분과 전원 최소 1개 담당이 끝날 때까지 미션·보스 차단
- 교사 전용: THE WAR 첫 QR 스캔 전 확정 파티 비상 재개방
- 장비 전투력 기준 랭킹
- 교환소 1 / 교환소 2
  - 각 교환소에 동시에 2명 입장
  - 각자 최대 2칸 장비 제시
  - 양쪽 동의 후 서버에서 동시에 교환
- QR 보상 중복 방지
  - 장비 뽑기 QR은 보상 횟수만큼 반복 사용
  - 미션·히든·최종 보상 QR은 학생별 1회만 지급
  - `qr_claims(player_id, qr_code)` unique 제약으로 중복 지급 차단
- 교사용 관리자 화면

## THE WAR 6가지 역할 가이드

`THE WAR`에서 파티 전원이 공동 QR 스캔을 마치고 공개 연출을 확인하면 다음 파트 카드가 표시됩니다. 각 장비 파트는 한 명만 맡을 수 있고, 파티원은 자신의 화면에서 직접 선택하거나 해제합니다.

| 장비 | 필요 능력 | 역할 가이드 | 잘 맞는 파티원 |
| --- | --- | --- | --- |
| 진리의 허리띠 | 판단력 | 관찰자 | 관찰과 판단이 빠른 사람 |
| 의의 호심경 | 인내력 | 수호자 | 집중력과 버티기가 강한 사람 |
| 복음의 신발 | 스피드 | 전령 | 달리기와 순발력에 자신 있는 사람 |
| 믿음의 방패 | 협동력 | 탱커 | 소통하며 팀을 지키는 데 강한 사람 |
| 구원의 투구 | 지력 | 전략가 | 퀴즈, 기억력, 퍼즐에 강한 사람 |
| 말씀의 검 | 힘 | 돌격대 | 힘과 신체 활동에 자신 있는 사람 |

한 사람은 최소 1개, 최대 2개 파트를 맡습니다. 6개 파트에 담당자가 모두 정해지고 모든 파티원이 최소 1개를 맡아야 배분이 완료됩니다. 선택 결과는 파티원 전체 화면에 실시간으로 반영되며, 완료 후 각 파트 담당자가 해당 미션의 대표로 나갑니다.

## 기술 구조

- 정적 프론트: `index.html`, `styles.css`, `app.js`
- Vercel API: `api/app.js`
- 서버 로직: `server/`
- DB: Postgres `DATABASE_URL`
  - Neon Postgres 권장
  - Supabase를 사용하는 경우 프로젝트의 Postgres connection string을 `DATABASE_URL`로 등록
  - Supabase Vercel 연동의 `POSTGRES_URL`도 자동 인식
- 로컬 개발 DB: `.data/dev-db.json`

서버 안전장치:

- PostgreSQL serializable transaction + advisory lock으로 쓰기 요청 직렬 처리
- 일시적인 연결·직렬화 오류 최대 3회 재시도
- DB 연결·쿼리 시간 제한과 64KB 요청 본문 제한
- 연결 중단 시 학생 화면에 서버 연결 안내를 표시하고 복구 후 자동 해제

Vercel/production에서는 `DATABASE_URL` 또는 `POSTGRES_URL`이 반드시 필요합니다. 둘 다 없으면 파일 DB로 대신 진행하지 않고 서버 오류를 반환합니다.

## 로컬 실행

저장소 루트에서 시작했다면 먼저 `current-retreat` 폴더로 이동합니다.

### Windows 간편 실행

이 PC에서만 테스트할 때는 `START_LOCAL_TEST.cmd`를 더블클릭합니다. 충돌 없는 포트로 서버가 시작되고 기본 브라우저가 자동으로 열립니다.

PowerShell에서는 실행 정책 문제를 피하도록 `npm` 대신 `npm.cmd`를 사용합니다.

```powershell
cd C:\Users\apdls\Desktop\bible1\current-retreat
npm.cmd run start:test
```

직접 실행하려면 다음 명령을 사용합니다.

```powershell
npm.cmd install
npm.cmd run dev
```

기본 주소는 `http://127.0.0.1:8765`입니다. `index.html`을 직접 열거나 Python `http.server`, VS Code Live Server로 열면 `/api/app` 서버가 없어 정상 동작하지 않습니다.

### 같은 Wi-Fi 휴대폰 테스트

`START_LAN_TEST.cmd`를 더블클릭하고 이번 실행에 사용할 교사 PIN을 정합니다. 또는 다음 명령을 사용합니다.

```powershell
$env:ADMIN_PIN="현장용-긴-PIN"
npm.cmd run start:lan
```

실행 창에 표시되는 `같은 Wi-Fi 휴대폰 접속` 주소를 휴대폰에서 엽니다. Windows 방화벽은 개인 네트워크에서만 허용하고, 행사 본운영은 Vercel HTTPS 배포를 사용합니다.

### 학생 입장과 교사 PIN

- 학생 개인 인증은 사용하지 않습니다. 이름과 조 이름이 기존 캐릭터와 같으면 자동으로 이어지고, 없으면 새 캐릭터가 생성됩니다.
- 교사 PIN에는 고정 기본값이 없습니다. LAN 실행 시 운영자가 입력하거나 Vercel의 `ADMIN_PIN`으로 직접 정합니다.
- `START_LOCAL_TEST.cmd`로 이 PC에서만 테스트할 때는 교사 PIN을 비워도 관리자 화면을 열 수 있습니다.

## DB와 운영 시간 설정

1. Neon 또는 Supabase에서 Postgres DB를 만듭니다.
2. SQL 콘솔에서 `schema.sql` 전체를 실행합니다.
3. Vercel 환경변수에 다음 값을 등록합니다.

```text
DATABASE_URL=postgresql://...
ADMIN_PIN=원하는관리자PIN
THE_WAR_OPENS_AT=2026-08-04T22:00:00+09:00
EVENT_ENDS_AT=2026-08-05T06:00:00+09:00
PG_POOL_MAX=2
```

### `THE_WAR_OPENS_AT`

- ISO-8601 날짜·시간으로 지정합니다. 한국 현장 시간은 반드시 `+09:00`을 포함해 입력합니다.
- 명시한 값이 있으면 그 시각을 `THE WAR` 개방 시각으로 가장 먼저 사용합니다.
- 값이 없고 `EVENT_ENDS_AT`만 있으면, 종료 시각이 속한 **한국 날짜의 00:00**에 자동으로 엽니다.
- 두 값이 모두 없으면 로컬 개발과 리허설 편의를 위해 즉시 `OPEN` 상태가 됩니다.

### `EVENT_ENDS_AT`

- 수련회 전체 기간을 추정해 넣는 값이 아니라 **실제 운영을 끝낼 시각**입니다.
- 지정한 시각부터 게임 API는 `410 EVENT_ENDED`를 반환하고 학생 화면은 운영 종료 안내로 전환됩니다.
- 한국 현장 시간은 반드시 `+09:00`을 포함해 명시합니다.

`PG_POOL_MAX` 기본값은 `2`입니다. 서버리스용 pooled `DATABASE_URL`을 사용하세요.

기존 DB에 적용할 때도 `schema.sql`을 다시 실행하면 됩니다. 필요한 테이블과 교환소 기본 데이터가 없으면 생성됩니다.

## 서버 상태 확인

`/api/app?action=health`에서 배포와 운영 시간을 확인합니다.

| 필드 | 의미 |
| --- | --- |
| `ok` | 상태 조회 성공 여부 |
| `service` | API 식별자 `current-retreat-api` |
| `status` | 운영 중 `ready`, 종료 후 `ended` |
| `active` | 전체 행사 API 운영 여부 |
| `endsAt` | 적용된 운영 종료 시각, 미설정이면 `null` |
| `warOpen` | 현재 `THE WAR`가 열렸는지 여부 |
| `warOpensAt` | 적용된 개방 시각, 즉시 개방이면 `null` |
| `storage` | `postgres` 또는 로컬 테스트용 `file` |
| `players` | 현재 저장된 학생 수 |
| `time` | 서버가 응답한 현재 시각 |

본운영의 기본 확인값은 `active: true`, `storage: postgres`입니다. 마지막 날 전에는 `warOpen: false`, 개방 시각 이후에는 `warOpen: true`인지 함께 확인합니다. `state` 응답에도 `warOpen`, `warOpensAt`이 포함되어 학생 화면의 잠금 상태를 결정합니다.

## QR URL 정책

앱은 clean URL을 기준으로 운영합니다. Vercel rewrite와 로컬 dev fallback이 설정되어 있어 휴대폰 기본 카메라로 다음 주소를 직접 열 수 있습니다.

- 홈: `/`
- 파티: `/party`
- 랭킹: `/ranking`
- 교환소: `/exchange/1`, `/exchange/2`
- 장비 뽑기: `/draw/1`, `/draw/2`, `/draw/3`
- `THE WAR` 공동 체크인 QR: `/team/merge`
- `THE WAR`: `/forest`
- 관리자: `/admin`
- 미션: `/mission/judgment`, `/mission/endurance`, `/mission/speed`, `/mission/teamwork`, `/mission/intellect`, `/mission/power`
- 히든: `/hidden/forest-cache-1`, `/hidden/forest-cache-2`
- 보스/최종 보급: `/boss`

운영 원칙:

- 장비 뽑기는 파티 명단 확정 후 이용합니다.
- 뽑은 장비는 학생의 개인 인벤토리에 보관하며, `THE WAR` 공동 체크인 전에는 합체 결과를 공개하지 않습니다.
- `/forest`는 개방 전에는 잠금 안내를, 개방 후 전원 스캔 전에는 `QR 스캔하기` 버튼만 표시합니다.
- 확정 명단의 모든 파티원이 `/team/merge` QR을 스캔해야 하며, 대기 화면은 완료 인원을 `N/N`으로 표시합니다.
- 마지막 스캔이 끝나면 장비·전투력 스냅샷을 생성하고 모든 파티원에게 합체 연출을 보여줍니다.
- 공개 연출 후 각 파티원이 6개 고유 파트 중 1~2개를 직접 맡습니다. 한 파트는 중복 선택할 수 없습니다.
- 6개 파트와 전원 최소 1개 담당 조건이 충족될 때까지 미션·보스 조회와 보상 지급을 차단합니다.
- 확정 파티는 학생·파티장이 변경할 수 없습니다. 조 편성 오류는 교사만 THE WAR 첫 QR 스캔 전 비상 재개방할 수 있고, 스캔이 시작되면 변경할 수 없습니다.
- 미션·히든·보스 보상은 학생별 1회만 지급하고 재스캔을 차단합니다.
- 기존 hash URL은 일부 호환하지만 새 QR은 clean URL로 생성합니다.

2. `active: true`, `storage: postgres`, `warOpensAt`, `endsAt`이 운영표와 일치하는지 확인합니다.
3. 첫 로그인 후 4명 미만일 때 파티 명단을 확정할 수 없는지 확인합니다.
4. 같은 조 이름으로 4~6명이 입장할 때 대기실 인원이 자동 갱신되는지 확인합니다.
5. 파티장이 명단을 확정하면 바로 장비 뽑기가 열리는지 확인합니다.
6. 같은 이름과 조 이름으로 다시 입장하면 기존 캐릭터가 이어지는지 확인합니다.
7. 여러 파티원이 각각 뽑은 장비가 개인 인벤토리에 남고, 공동 QR 스캔 전에는 합체 결과가 공개되지 않는지 확인합니다.
8. 명단 확정부터 장비 뽑기까지 추가 준비 단계 없이 이어지는지 확인합니다.
9. `THE WAR` 개방 전 `/forest`에서 잠금 안내와 개방 시각이 표시되는지 확인합니다.
10. 개방 시각 후 `/forest`에 `QR 스캔하기` 버튼 하나만 표시되는지 확인합니다.
11. 확정된 파티원 전원이 같은 `/team/merge` QR을 스캔하고, 각 화면의 대기 인원이 `N/N`으로 동기화되는지 확인합니다.
12. 마지막 파티원이 스캔하면 모든 화면에서 합체 연출과 장비·전투력 계산 결과가 공개되는지 확인합니다.
13. 공개 후 각 파티원이 자신의 화면에서 1~2개 파트를 선택·해제하고, 같은 파트를 중복 선택할 수 없는지 확인합니다.
14. 파트 선택이 파티원 전체 화면에 동기화되고, 6개 파트와 전원 최소 1개 담당 조건을 모두 충족해야 완료되는지 확인합니다.
15. 파트 배분 완료 전 미션·보스 URL 직접 접근과 보상 API가 차단되는지 확인합니다.
16. 확정 뒤 학생·파티장이 명단을 다시 열 수 없는지, 교사만 THE WAR 첫 QR 스캔 전 비상 재개방할 수 있는지 확인합니다. 첫 QR 스캔 뒤에는 재개방이 차단되어야 합니다.
17. 미션·히든·보스 보상이 첫 스캔에만 지급되고 재스캔이 차단되는지 확인합니다.
18. 교환소 1/2에서 두 학생의 입장, 선택, 동의, 완료와 양쪽 인벤토리 갱신을 확인합니다.
19. 서버를 잠시 중단했을 때 연결 안내가 나타나고 복구 후 사라지는지 확인합니다.
20. 관리자 화면에서 학생, 랭킹, 교환소, QR 링크, 장비 조정 기능을 확인합니다.
21. 테스트 환경에서 `EVENT_ENDS_AT` 이후 API가 `410 EVENT_ENDED`를 반환하는지 확인합니다.
22. 현장 교사 휴대폰과 학생 휴대폰의 Safari·Chrome에서 실제 Vercel URL을 확인합니다.

## 운영 주의사항

- 학생 개인 인증을 사용하지 않으므로 이름과 조 이름을 정확하고 일관되게 입력하도록 안내합니다.
- 조 이름은 모든 파티원이 띄어쓰기까지 똑같이 입력해야 합니다.
- 파티 명단은 최소 4명, 최대 6명입니다. 명단을 확인한 뒤 파티장이 확정합니다.
- 마지막 날에는 확정 명단의 모든 파티원이 같은 `THE WAR` QR을 각자 스캔해야 합니다.
- 파트는 학생이 자신의 화면에서 직접 선택합니다. 한 사람당 1~2개, 한 파트당 한 명이며 모든 선택은 파티 전체에 동기화됩니다.
- 확정 명단은 학생·파티장이 취소할 수 없습니다. 편성 오류는 교사만 THE WAR 첫 QR 스캔 전 비상 재개방할 수 있으며, 첫 스캔 뒤에는 영구적으로 변경할 수 없습니다.
- 교사 PIN은 짧은 숫자보다 추측하기 어려운 긴 문자열을 사용합니다.
- 교환소는 선생님 앞에서 줄을 세워 운영하는 것을 전제로 합니다.
- 랭킹은 WebSocket이 아니라 3~5초 polling 방식입니다.
- 교환소 입장 상태는 10분 후 자동 만료되어 다음 요청 시 초기화됩니다.
- 본운영 전에는 실제 Vercel URL, 실제 DB, 실제 QR 인쇄물로 전체 흐름을 리허설합니다.
